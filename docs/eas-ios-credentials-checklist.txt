EAS iOS signing checklist (implements plan: Development vs production)
======================================================================

Bundle IDs (from app.config.ts):
  - Development: com.codeandtheory.templatepipelinetest.dev  (EAS profile: development)
  - Production:  com.codeandtheory.templatepipelinetest.dist (EAS profile: production, distribution: store)

expo.dev — com.codeandtheory.templatepipelinetest.dev
------------------------------------------------------
1. Under the Development tab: use an Apple Development certificate (not Apple Distribution).
2. Add an iOS App Development provisioning profile for this bundle ID (table must not be empty).
3. Register devices in Apple Developer and include them in the development profile.
4. Wrong type fix: remove a Distribution certificate from the Development tab if one was uploaded there.

expo.dev — com.codeandtheory.templatepipelinetest.dist
------------------------------------------------------
1. Under the App Store tab: Apple Distribution certificate + App Store provisioning profile (both required).
2. If Team shows as None, reconnect Apple (Apple ID login or App Store Connect API key in Expo).

Bootstrap for non-interactive CI
--------------------------------
1. From this repo, run once with the same Expo account as EXPO_TOKEN:
     eas build --platform ios --profile development
   or: eas credentials -p ios
2. Then re-run GitHub Actions EAS iOS jobs (--non-interactive).

Firebase / consoles (manual)
-----------------------------
After switching production to .dist, register the new Android applicationId and iOS bundle ID in
Firebase and Google Play / App Store Connect as needed; update GitHub secrets if you add prod Firebase apps.
